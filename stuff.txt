To add media items (single or bulk), we will use the Modern Android Photo Picker (which now supports audio) or the standard Document Picker. In 2026, the GetMultipleContents contract is the most reliable way to let users pick several audio files at once.
We need to follow the MVVM pattern here:

The UI launches the picker.
The ViewModel receives the URIs and processes the metadata (Title, Artist, Duration).
The Database saves the results.
1. The Metadata Helper
Extracting metadata from a file URI is tricky. We'll create a small utility function to handle this using MediaMetadataRetriever.
Kotlin

// In a new file: util/MetadataHelper.ktfun getMediaMetadata(context: Context, uri: Uri): MediaEntity {
    val retriever = MediaMetadataRetriever()
    return try {
        retriever.setDataSource(context, uri)
        val title = retriever.extractMetadata(MediaMetadataRetriever.METADATA_KEY_TITLE) ?: "Unknown Title"
        val artist = retriever.extractMetadata(MediaMetadataRetriever.METADATA_KEY_ARTIST) ?: "Unknown Artist"
        val duration = retriever.extractMetadata(MediaMetadataRetriever.METADATA_KEY_DURATION)?.toLong() ?: 0L
        
        MediaEntity(
            uri = uri.toString(),
            title = title,
            creator = artist,
            duration = duration,
            fileName = uri.lastPathSegment ?: "unknown_file",
            dateAdded = System.currentTimeMillis(),
            mimeType = context.contentResolver.getType(uri),
            artworkUri = null // We can handle artwork extraction later
        )
    } catch (e: Exception) {
        // Fallback if metadata is corrupt
        MediaEntity(uri.toString(), "Unknown", "Unknown", 0L, "file", System.currentTimeMillis(), null, null)
    } finally {
        retriever.release()
    }
}
2. The HomeViewModel
This is where the logic lives. We inject the MediaDao we created earlier.
Kotlin

@HiltViewModelclass HomeViewModel @Inject constructor(
    private val mediaDao: MediaDao,
    @ApplicationContext private val context: Context
) : ViewModel() {

    // Get all media from DB as a Flow for the UI to observe
    val allMedia = mediaDao.getAllMedia().stateIn(
        scope = viewModelScope,
        started = SharingStarted.WhileSubscribed(5000),
        initialValue = emptyList()
    )

    fun importMedia(uris: List<Uri>) {
        viewModelScope.launch(Dispatchers.IO) {
            val entities = uris.map { uri ->
                // Ensure we have persistent permission to access this file later
                context.contentResolver.takePersistableUriPermission(
                    uri, Intent.FLAG_GRANT_READ_URI_PERMISSION
                )
                getMediaMetadata(context, uri)
            }
            mediaDao.insertMediaList(entities)
        }
    }
}

@Composablefun HomeScreen(

⚠️ Important: Permissions Update
Since you are selecting files from the device, you need to tell Android you intend to read them. Even with the picker, for an "Offline Player" to work in the background, you should add this to your AndroidManifest.xml:
XML

<uses-permission android:name="android.permission.READ_MEDIA_AUDIO" />
🔍 How this works:
When the user clicks Add Media, the system opens the file manager.
The user can long-press to select multiple MP3s.
Once they hit "Select," the URIs are sent to the HomeViewModel.
The ViewModel runs a background task (Coroutine) to extract the song titles and durations.
takePersistableUriPermission is used so that when the user reboots their phone, your app still has permission to play those specific files.
